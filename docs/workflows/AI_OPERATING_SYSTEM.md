# Varada Nexus AI Operating System

## Purpose

This document defines how AI agents will help plan, build, test, publish, and maintain the Varada Nexus website, EMS, tools, blogs, SEO pages, and service pages.

## Core Rule

AI should assist development, planning, SEO, and content creation.

AI must NOT be integrated into free public tools unless explicitly approved.

## System Layers

### 1. Executive Layer

Responsible for project direction and final decisions.

Agents:
- Master Agent
- Project Manager Agent
- Quality Controller Agent

### 2. Strategy Layer

Responsible for planning what to build.

Agents:
- Tool Strategy Agent
- SEO Strategy Agent
- Blog Strategy Agent
- Service Strategy Agent
- Lead Generation Agent

### 3. Development Layer

Responsible for implementation.

Agents:
- Frontend Developer Agent
- Supabase Agent
- Database Agent
- GitHub Deployment Agent

### 4. Content Layer

Responsible for content assets.

Agents:
- Blog Writer Agent
- SEO Page Agent
- FAQ Agent
- Schema Agent
- CTA Agent

### 5. QA Layer

Responsible for testing and quality control.

Agents:
- Formula QA Agent
- SEO QA Agent
- UI QA Agent
- Code Review Agent
- Publishing QA Agent

## Standard Workflow for Every Tool

1. Tool Strategy Agent selects and prioritizes the tool.
2. Formula Agent defines inputs, assumptions, formulas, and output.
3. SEO Page Agent creates SEO content.
4. UI/UX Agent designs the layout.
5. Frontend Developer Agent builds HTML, CSS, and JavaScript.
6. QA Agent tests calculation, UI, mobile view, SEO, and CTA.
7. Publishing Agent prepares deployment, sitemap, and internal links.

## Standard Workflow for Every Blog

1. Blog Strategy Agent chooses topic.
2. SEO Agent defines keyword and search intent.
3. Blog Writer Agent writes content.
4. Internal Linking Agent connects blogs, tools, and services.
5. SEO QA Agent checks title, meta, headings, schema, and links.
6. Publishing Agent prepares upload.

## Standard Workflow for Every Service Page

1. Service Strategy Agent defines target audience.
2. SEO Agent defines keyword and page structure.
3. Content Agent writes service content.
4. CTA Agent adds conversion sections.
5. Schema Agent adds structured data.
6. QA Agent reviews.
7. Publishing Agent prepares deployment.

## Technology Rules

Frontend:
- HTML
- CSS
- JavaScript

Backend:
- Supabase Edge Functions
- TypeScript
- Deno

Database:
- Supabase PostgreSQL

Hosting:
- GitHub Pages

Rules:
- Prefer static/client-side tools.
- Avoid paid APIs.
- Keep pages fast.
- Use reusable components.
- Maintain clean folder structure.
- Follow SEO best practices.
- Add professional disclaimers where needed.

- Free consultation

- Project estimate
- BOQ/DPR support
- Compliance support
- Execution support

Every output must be:
- Professional

- Mobile-friendly
- SEO-friendly
