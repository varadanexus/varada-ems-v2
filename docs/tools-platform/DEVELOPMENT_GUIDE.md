# Tools Platform Development Guide

## Purpose

This guide defines how new Tools Platform initiatives should be documented and prepared before implementation.

## Tool Requirements

- Must solve one clear problem.
- Must use rules, formulas, or structured logic.
- Must have a measurable SEO intent.
- Must support a relevant service CTA.
- Must be understandable without expert training.

## Recommended Workflow

1. Select a sector.
2. Identify a narrow user problem.
3. Define the tool category.
4. Write the specification.
5. Define formula or rule logic.
6. Define page content and SEO structure.
7. Define CTA path.
8. Review with QA checklist.

## Documentation Workflow

- Create or update a file under `docs/tools-platform/specifications/`.
- Use templates from `docs/tools-platform/templates/`.
- Use agents from `docs/tools-platform/agents/` for structured role-based drafting.

## EMS References

Where platform work touches existing EMS context, reference by path only.

Examples:

- `docs/CURRENT_STATE_FOR_AI.md`
- `docs/architecture/ARCHITECTURE_MAP.md`
