# Varada Nexus Definition of Done (DoD)

This document defines the minimum quality standard for every deliverable produced within the Varada Nexus Tools Platform.

No task is considered complete until every applicable requirement has been satisfied.

---

# Universal Requirements

Every deliverable must:

- Be accurate.
- Be implementation-ready.
- Be professionally written.
- Follow project standards.
- Avoid duplication.
- Be maintainable.
- Be reusable whenever possible.
- Be documented.

---

✔ Target audience

✔ Industry sector

✔ User problem being solved

✔ Output format

✔ Limitations

## Quality Assurance

## Lead Generation

Every tool must naturally guide the user toward:

- Consultation
- Detailed report
- Execution services
- Planning services
- Implementation support

The CTA must feel helpful, not sales-driven.

---

# Documentation

Every completed tool must produce:

- Specification
- Development notes
- Testing checklist
- Future improvements

---

# Completion Rule

If any required section is missing, the task is NOT complete.

Claude should continue refining the deliverable until every requirement has been satisfied.
