# Varada Nexus Execution Workflow

This document defines how every request should be executed.

Claude should NEVER jump directly into implementation.

Every task must pass through the following stages.

---

# Phase 1 — Understanding

Before creating anything:

Understand:

- Business objective
- User objective
- Target industry
- Target audience
- Expected outcome

If information is missing,

Ask questions first.

Never assume.

---

# Phase 2 — Research

Research:

- Industry standards
- Existing competitors
- Best practices
- User expectations
- SEO opportunity
- Technical feasibility

Identify risks.

---

# Phase 3 — Planning

Create:

- Product requirements
- Functional requirements
- Technical requirements
- UX requirements
- SEO requirements

Review before continuing.

---

# Phase 4 — Specification

Produce:

Complete Tool Specification

using

TOOL_SPECIFICATION_TEMPLATE.md

Nothing should be implemented before the specification is approved.

---

# Phase 5 — Internal Review

Review the specification from:

Product

Engineering

SEO

UX

QA

Business

Revise until all reviewers agree.

---

# Phase 6 — Implementation Planning

Only after approval:

Create:

- HTML plan
- CSS plan
- JavaScript plan
- Folder structure
- Component structure
- File organization

Do NOT start coding until the implementation plan is complete.

---

# Phase 7 — Implementation

Generate production-ready code.

Code must be:

Clean

Reusable

Documented

Responsive

Accessible

Fast

---

# Phase 8 — Quality Review

Review:

Functionality

Performance

Accessibility

SEO

Code Quality

Security

User Experience

Cross-browser compatibility

---

# Phase 9 — Documentation

Generate:

Developer notes

Testing checklist

Future improvements

Known limitations

---

# Phase 10 — Final Approval

Only mark complete when:

Business ✓

Engineering ✓

SEO ✓

QA ✓

Documentation ✓

All complete.

Never skip phases.

Never jump directly to coding.
